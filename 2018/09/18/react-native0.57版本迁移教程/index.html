react-native版本迁移教程

发表于 2018-09-18 | 分类于 react-native

如果在学习react-native的过程中遇到什么问题，欢迎加入QQ群397885169一起学习，一起成长。

前言

react-native在三天前，也就是2018.9.12号更新了0.57.0版本，这个版本终于正常的兼容了babel7，解决了0.56版本使用mobx等装饰器需要各种配置的问题。

主要更新（合并0.56版本和0.57版本重要的修复）

  • TextInput不能输入中文的问题

  • TextInput.clear()和TextInput.setNativeProps({text: ''})不能正常使用的问题

  • 更新iOS最低兼容到iOS9
  • 为WebView添加WKWebView，通过useWebKit={true}开启
  • 为安卓添加overflow
  • 为安卓Image添加默认占位图defaultSource，但是在debug下是看不到的，只有在release中才能看到
  • 安卓现在TextInput的underlineColorAndroid默认为透明(终于不用手动更改了)
  • 打开Modal可以覆盖状态栏层？
  • 修复Modal+ FlatList滚动
  • 导出FlatList和SectionList的动画，
    现在支持Animated的组件包括了View，Text，Image，ScrollView，FlatList，SectionList。

以上是0.56和0.57版本，我整理的主要更新，更详细的更新日志，请参考官方日志。

项目升级

升级

开发react-native经常会遇到升级新版本，官方提供了命令行的升级方式，我管这种叫做升级，这种方式的好处就是傻瓜式，执行命令行就可以自动升级版本了，但这个方式的缺陷也很明显，如果升级中出现问题，很难排查错误。所以，我更推荐第二种方式。

迁移

迁移：顾名思义，搬迁移动，流程就是，新建个项目，把原有项目中的更改，慢慢搬家，搬到新的项目中，这样的坏处很明显，很慢，才做过程中，可能会有遗忘，但是这样方式，更方便管理，流程出现错误，只要对着原项目更改就可以了，更可控。

0.57版本迁移教程

未使用装饰器

如果你的项目中，没有用到Mobx，这种需要es6装饰器语法支持的三方库或者方法，那么恭喜你，只要你完整的迁移项目就可以了，除了YellowBox会给你造成一点麻烦外，基本不会遇到其他问题的，如果真的遇到问题啦，欢迎加群，咱们一起讨论解决办法。

使用装饰器

如果你的项目使用了装饰器，那么也要恭喜你，顺利的度过了0.56版本的那个深坑，在0.57版本中，要修改的东西相对于0.56少了很多。

小技巧

新版本中，对YellowBox做了增强，项目中有各种问题都会暴露出来，可以使用YellowBox.ignoreWarnings(['Require cycle:'])关闭一波。
